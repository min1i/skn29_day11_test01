### Day 34 — 분류 평가지표 심화 & 불균형 데이터 실전

#### 분류 평가지표 (Day34_01)
- **Confusion Matrix**: TP / TN / FP / FN 4가지 케이스
- **Precision (정밀도)**: 모델이 Positive로 예측한 것 중 실제 Positive 비율 → 틀리면 안 되는 상황(스팸 메일)
  $$\text{Precision} = \frac{TP}{TP + FP}$$
- **Recall (재현율)**: 실제 Positive 중 모델이 맞게 찾아낸 비율 → 놓치면 안 되는 상황(암 진단)
  $$\text{Recall} = \frac{TP}{TP + FN}$$
- **F1-Score**: Precision과 Recall의 조화평균 (트레이드오프 균형)
  $$F1 = 2 \times \frac{\text{Precision} \times \text{Recall}}{\text{Precision} + \text{Recall}}$$
- **F_β**: β > 1이면 Recall 중시, β < 1이면 Precision 중시
- **Precision-Recall Curve**: threshold 변화에 따른 두 지표의 트레이드오프 시각화
- **ROC Curve**: TPR(Recall, y축) vs FPR(x축) 곡선
  - FPR = FP / (FP + TN)
- **AUC**: ROC 곡선 아래 면적 — 1에 가까울수록 좋은 모델

**Recall 향상 전략 2가지**:
1. threshold를 낮춰서 Recall 값을 올리는 임계값 탐색
2. `BCEWithLogitsLoss(pos_weight=...)` — 양성 클래스 놓칠 때 패널티 강화

#### 불균형 데이터 실전: Credit Card Fraud Detection (Day34_02)
- 데이터: 정상 99.83% / 사기 0.17% (284,807건)
- 핵심 지표: **Recall** (사기 놓치면 고객 피해)
- **pos_weight 계산**: `num_neg / num_pos` — 불균형 비율만큼 Positive 클래스에 가중치 부여
- `DataLoader` 배치 학습 (batch_size=2048), 20 에포크

### Day 35 — 회귀 평가지표 이론 & XGBoost

#### 회귀 평가지표 (Day35_01)
- **MAE**: 모든 오차에 동일 가중치, 이상치에 덜 민감, 직관적
  $$MAE = \frac{1}{n} \sum |y_i - \hat{y}_i|$$
- **MSE**: 오차 제곱 → 큰 오차에 강한 패널티 (금융 리스크, 의료 진단 도메인)
  $$MSE = \frac{1}{n} \sum (y_i - \hat{y}_i)^2$$
- **RMSE**: MSE에 루트 → 원래 데이터 스케일 복원 + MSE 장점 유지
  $$RMSE = \sqrt{MSE}$$
- **R²**: 모델이 데이터 분산을 설명하는 상대적 성능 (0~1)
  $$R^2 = 1 - \frac{SSE}{SST}$$
  - SSE: 잔차 제곱합 / SST: 실제값의 편차 제곱합

#### XGBoost 첫 등장
- `XGBRegressor`: Iris 회귀 R² = **0.919**
- DNN 회귀 (BatchNorm + Dropout + `PolynomialLR` 스케줄러): R² = **0.939**
